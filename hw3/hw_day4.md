### Summary

- **Навігаційні властивості (Navigation Properties)**: Це спосіб EF Core перетворити `JOIN` у SQL на зручні об'єкти в C#. Вони бувають двох типів: Reference (посилання на один об'єкт) та Collection (список об'єктів).
- **Складні зв'язки**:
    - **1:1 (One-to-One)**: Використовується для винесення важких або опціональних даних в окрему таблицю (наприклад, `Course` та `CourseDetail`).
    - **M:N (Many-to-Many)**: EF Core вміє створювати проміжну таблицю автоматично (Implicit Join Table), якщо ви просто додасте колекції з обох сторін (`Student` має `List<Course>`, а `Course` має `List<Student>`).
- **Міграції (Code-First)**: Це система контролю версій для вашої бази даних. Змінили C# клас → створили міграцію → оновили базу. У міграції можна додавати свій власний SQL (`migrationBuilder.Sql()`), якщо можливостей EF Core не вистачає (наприклад, для `VIEW` або тригерів).
- **Eager Loading (`Include`)**: Зв'язані дані не завантажуються з бази автоматично (щоб економити пам'ять). Якщо вам потрібен викладач разом з його курсами, ви **зобов'язані** явно вказати це через `.Include(i => i.Courses)`.
